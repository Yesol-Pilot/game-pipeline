# 🌌 Game Pipeline: The Singularity Expansion
**"자동화를 넘어, 자율화(Autonomy)로."**

우리는 게임을 '만드는' 도구를 완성했습니다.
이제 시스템 스스로 게임을 '검증'하고, '개선'하고, '진화'시키는 단계로 나아갑니다.

## 🤖 Phase 16: Autonomous QA (AI Playtesting)
**"버그는 사람이 잡는 것이 아닙니다. AI가 잡습니다."**
*   **Reinforcement Learning (RL)**: Godot RL Agents를 연동하여 AI가 24시간 게임을 플레이합니다.
*   **Balance Heatmap**: AI가 자주 죽는 구간을 히트맵으로 시각화하여 레벨 디자이너 없이 난이도를 자동 조절합니다.
*   **Crash Hunter**: 예외 상황(벽 뚫기, 무한 추락)을 찾아내어 리포트를 생성합니다.

## 🧠 Phase 17: Neuro-Marketing (Feedback Loop)
**"시장의 목소리가 코드로 변환됩니다."**
*   **Review Sentiment Analysis**: 앱스토어/스팀 리뷰를 크롤링하여 긍정/부정 감성을 분석합니다.
*   **Auto-Patching**: "조작감이 너무 뻑뻑해요"라는 리뷰가 많으면, 시스템이 자동으로 `player_speed`와 `friction` 값을 조정한 패치 버전을 제안합니다.

## 🧬 Phase 18: Self-Healing Codebase
**"코드가 스스로를 고칩니다."**
*   **Error Self-Correction**: 런타임 에러 로그를 LLM에게 전송 -> LLM이 원인 분석 -> 깃허브 PR 자동 생성.
*   **Legacy Refactoring**: 사용되지 않는 에셋과 코드를 주기적으로 감지하여 정리(GC)합니다.

---
## 🎯 Recommendation
**[Phase 16: Autonomous QA]**가 가장 시급하고 효과적입니다.
만들어진 게임의 품질을 사람이 일일이 테스트하는 병목을 제거할 수 있기 때문입니다.
